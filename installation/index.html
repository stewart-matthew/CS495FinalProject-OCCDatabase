Installation Guide

This document explains how to install and deploy the OCC Database application.

Requirements

Before getting started, ensure you have the following:

  • Node.js – JavaScript runtime for running the web app locally
  • Supabase Account – Free database and authentication service
  • Git – For cloning the repository
  • Docker (optional) – For containerized deployment
  • GitHub Account (optional) – For Render deployments

1. Running the Web App Locally

Step 1: Clone the Repository

git clone https://github.com/stewart-matthew/CS495FinalProject-OCCDatabase.git
cd CS495FinalProject-OCCDatabase

Step 2: Install Node Dependencies

npm install

This installs React, Supabase JS, Tailwind CSS, and other dependencies.

Step 3: Add Environment Variables

Create a .env file in the root directory with your Supabase credentials:

REACT_APP_SUPABASE_URL=your_supabase_url
REACT_APP_SUPABASE_ANON_KEY=your_supabase_anon_key

Step 4: Run the Development Server

npm start

The web app will automatically open at http://localhost:3000

2. Running with Docker (Recommended)

Docker provides a consistent, containerized environment for deployment.

Step 1: Build the Docker Image

docker build -t stewartmatthew6288/occ-site .

Step 2: Push to Docker Hub (Optional)

docker push stewartmatthew6288/occ-site

Step 3: Run the Docker Container

docker run -p 3000:80 stewartmatthew6288/occ-site

Access the application at http://localhost:3000

3. Hosting on Render (Recommended)

Render provides free static site and web service hosting with automatic deployments from GitHub.

Prerequisites

  • GitHub repository connected to Render
  • Environment variables configured in Render dashboard

Deployment Steps

Step 1: Build the React App Locally (Optional – Verify Locally)

npm run build

This creates a build/ folder with optimized static files.

Step 2: Connect to Render

  1. Go to dashboard.render.com
  2. Click "New +" → select "Static Site"
  3. Connect your GitHub account and select the repository
  4. Configure the following:
Setting Value
Build Command npm ci && npm run build
Publish Directory build
Environment Variables See below

Step 3: Add Environment Variables

In the Render dashboard, add the following environment variables:

  • REACT_APP_SUPABASE_URL = Your Supabase project URL
  • REACT_APP_SUPABASE_ANON_KEY = Your Supabase anonymous key

Step 4: Deploy

Click "Create Static Site" – Render automatically builds and deploys your application with each GitHub push.

Render Free Tier Limits

Feature Limit
Uptime 750 hours/month (~24/7 for one service)
Bandwidth 100 GB/month
SSL Certificates Automatic & free
Custom Domains Available

Deployment Best Practices

  • Security: Always add sensitive keys in Render's dashboard, never commit them to GitHub
  • Auto-Deploy: Enabled by default – pushes to main trigger new deployments
  • Custom Domains: Configure in Render dashboard under site settings
  • Preview Deployments: Available with paid plans for pull requests

Troubleshooting Render Deployments

Problem Solution
Build fails Check build logs in Render dashboard; verify npm run build works locally
Missing environment variables Ensure all required variables are set in Render dashboard
Static site not updating Clear Render cache; verify build/ folder is generated
Docker issues Test Dockerfile locally with docker build before deploying
